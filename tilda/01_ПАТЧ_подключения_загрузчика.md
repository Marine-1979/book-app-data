# ПАТЧ ПОДКЛЮЧЕНИЯ ЕДИНОГО ЗАГРУЗЧИКА К `/navigator`

Версия 1.0 · 18.07.2026

## 0. Что добавить

Создайте на `/navigator` новый T123 **самым первым блоком страницы** и вставьте в него содержимое:

`00_BAPP_DATA_LOADER_T123.html`

Загрузчик:
- один раз получает единый `dist/app-data.json`;
- передает один и тот же Promise всем T123-блокам;
- использует кеш браузера;
- при недоступности jsDelivr пробует GitHub Raw;
- при недоступности обоих источников использует встроенную последнюю рабочую копию.

После этого в трех существующих скриптах меняются только фрагменты загрузки данных.

---

## 1. Блок «Дашборд»

Удалите две константы:

```js
const ROUTES_URL = '...routes.json...';
const MATERIALS_URL = '...materials.json...';
```

Замените блок:

```js
Promise.all([
  fetch(ROUTES_URL).then(...),
  fetch(MATERIALS_URL).then(...)
]).then(([routeData, matData]) => {
  ROUTES    = routeData.routes  || [];
  MATERIALS = matData.materials || [];
  META      = matData.meta      || null;
```

на:

```js
window.BAPP_DATA.get().then(data => {
  ROUTES    = data.routes    || [];
  MATERIALS = data.materials || [];
  META      = data.meta      || null;
```

Остальную часть `.then(...)` и существующий `.catch(...)` не менять.

---

## 2. Блок «Навигатор маршрутов»

Удалите две константы:

```js
const ROUTES_URL = '...routes.json...';
const MATERIALS_URL = '...materials.json...';
```

Функцию `loadMaterials()` замените целиком:

```js
function loadMaterials() {
  if (MATERIALS) return Promise.resolve(MATERIALS);

  return window.BAPP_DATA.get().then(data => {
    MATERIALS = data.materials || [];
    return MATERIALS;
  });
}
```

Замените начальную загрузку:

```js
Promise.all([
  fetch(ROUTES_URL).then(...),
  fetch(MATERIALS_URL).then(...)
])
  .then(([routeData, matData]) => {
    ROUTES = routeData.routes || [];
    const materialsList = matData.materials || [];
```

на:

```js
window.BAPP_DATA.get()
  .then(data => {
    ROUTES = data.routes || [];
    const materialsList = data.materials || [];
```

Остальную часть `.then(...)` и `.catch(...)` не менять.

---

## 3. Блок «Инструменты»

Удалите константу:

```js
const MATERIALS_URL = '...materials.json...';
```

Замените:

```js
fetch(MATERIALS_URL)
  .then(r => {
    if (!r.ok) throw new Error(r.status);
    return r.json();
  })
  .then(data => {
    allMaterials = data.materials || [];
```

на:

```js
window.BAPP_DATA.get()
  .then(data => {
    allMaterials = data.materials || [];
```

Существующий `.catch(...)` оставить.

---

## 4. Проверка после публикации

1. Откройте `/navigator` в режиме инкогнито.
2. В DevTools → Network отфильтруйте `app-data`.
3. В нормальном режиме должна загружаться одна основная выдача:
   `dist/app-data.json`.
4. Откройте маршрут, «Последние семь» и «Смотреть инструменты».
5. Убедитесь, что повторных запросов `routes.json` и `materials.json` нет.
6. В консоли выполните:

```js
window.BAPP_DATA.get().then(data => console.log(
  data.routes.length,
  data.materials.length,
  data.build
));
```

Ожидаемо: `8`, `37`, объект `build`.

## 5. Отдельная обязательная правка шрифтов

В опубликованном `/navigator` сейчас есть локальные `@font-face` в коде страницы.
Так как приложение находится в том же проекте Tilda и получает шрифты из
глобального «MM · Фундамент», этот блок нужно удалить из настроек страницы.
Загрузчик шрифты не подключает.
