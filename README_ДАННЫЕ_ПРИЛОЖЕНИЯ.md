# Работа с данными цифрового приложения

## Принцип

Редактор продолжает работать с двумя понятными файлами:

- `data/materials.json` — материалы;
- `data/routes.json` — названия и описания маршрутов.

Приложение не читает их по отдельности. Автоматизация проверяет оба файла и
собирает один технический файл:

- `dist/app-data.json` — единственный файл, который загружает `/navigator`.

`dist/app-data.json` вручную не редактируется.

## Что означают папки

### `schema/`

Формальные правила устройства JSON: какие поля разрешены, какой у них тип,
какие значения допустимы. Это не контент и не новая база данных.

Текущая schema — совместимый черновик v0.1. Она сохраняет действующие поля и
уже предусматривает будущие `tags`, `tasks`, `published_at`, `updated_at`,
`parent_id`, не заставляя заполнять их сейчас.

### `scripts/`

Два небольших служебных сценария без внешних библиотек:

- `validate-data.mjs` — проверяет ошибки и предупреждения;
- `build-app-data.mjs` — объединяет данные и автоматически пересчитывает
  количество материалов в маршрутах.

### `.github/workflows/`

Здесь находится GitHub Actions — встроенный в GitHub «робот».

После изменения JSON он:
1. проверяет файлы;
2. прекращает публикацию при критической ошибке;
3. собирает новый `dist/app-data.json`;
4. коммитит только сгенерированный runtime-файл.

Его не нужно запускать вручную при каждом обновлении.

### Почему сейчас нет `snapshots/`

Отдельная папка снимков на первом этапе избыточна:

- все старые версии уже хранит история Git;
- последнюю рабочую копию хранит кеш браузера;
- резервная копия встроена в первый T123-блок загрузчика.

`snapshots/` понадобится только при отдельной автоматической публикации релизов
или при переносе резервной копии из Tilda в самостоятельное хранилище.

## Редакторский процесс после внедрения

1. Изменить `materials.json` и при необходимости `routes.json`.
2. Сохранить изменения в GitHub.
3. Дождаться зеленой проверки Actions.
4. Tilda не редактировать: `/navigator` получит новый `dist/app-data.json`
   автоматически после обновления CDN-кеша.

## Важное ограничение

`count` в `routes.json` больше не считается надежным источником. При сборке он
всегда вычисляется по фактическим материалам. В будущем поле можно удалить из
редакторского файла, но на первом этапе оно сохранено для обратной совместимости.
